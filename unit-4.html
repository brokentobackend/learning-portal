1. List and explain the secure communication protocols. Describe briefly about a firewall and its role in network security.

Secure Communication Protocols

Secure communication protocols ensure confidentiality, integrity, authentication, and secure data transfer over networks.

  1. SSL (Secure Sockets Layer)
    Encrypts data between client and server. Mostly replaced by TLS.
  2. TLS (Transport Layer Security)
    Advanced and more secure version of SSL. Used in HTTPS, email, and messaging.
  3. HTTPS (HTTP Secure)
    Secure version of HTTP using SSL/TLS for encrypted web communication.
  4. SSH (Secure Shell)
    Provides secure remote login and command execution.
  5. IPsec (Internet Protocol Security)
    Secures IP communication using encryption and authentication.
  6. SFTP (Secure File Transfer Protocol)
    Secure file transfer using SSH.

Firewall and Its Role in Network Security

A firewall is a security device or software that monitors and controls incoming and outgoing network traffic based on predefined security rules.

Role of Firewall

  • Blocks unauthorized access
  • Prevents malicious traffic
  • Enforces security policies
  • Acts as a barrier between trusted and untrusted networks

6. Write a brief note on Intrusion Prevention System (IPS). Explain the benefits and limitations of the stateful inspection technique in network security.

Intrusion Prevention System (IPS)

An Intrusion Prevention System (IPS) is a network security device or software that monitors network traffic in real time, detects malicious activities, and actively blocks or prevents attacks.

Functions of IPS

  • Inspects incoming and outgoing traffic
  • Detects known attacks using signatures and patterns
  • Prevents intrusions by dropping malicious packets
  • Protects against malware, DoS attacks, and exploits

Importance:
IPS provides active protection by stopping threats before they reach critical systems.

Stateful Inspection Technique

Stateful inspection is a firewall technique that tracks the state of active network connections and makes filtering decisions based on context.

Benefits

  • More secure than stateless filtering
  • Understands session state
  • Blocks unsolicited traffic
  • Improves protection against spoofing

Limitations

  • Requires more memory and processing power
  • Less effective against application-layer attacks
  • Performance may degrade under heavy traffic

7. Explain IPsec (Internet Protocol Security) and its working in securing network communications.

IPsec (Internet Protocol Security)

IPsec (Internet Protocol Security) is a suite of protocols used to secure IP communication by providing encryption, authentication, and integrity at the network layer.

Services Provided by IPsec

  • Confidentiality
  • Integrity
  • Authentication
  • Anti-replay protection

Working of IPsec

  1. IPsec establishes a Security Association (SA) between two devices
  2. Key exchange is performed using IKE (Internet Key Exchange)
  3. Data packets are encrypted and authenticated
  4. Secure communication is achieved using:
    • AH (Authentication Header) – integrity and authentication
    • ESP (Encapsulating Security Payload) – encryption and integrity

Modes of Operation

  • Transport Mode – protects payload only
  • Tunnel Mode – protects entire IP packet (used in VPNs)

Stateful Inspection in Network Security

Stateful inspection is a firewall technique that tracks the state of active connections and allows or blocks traffic based on the context of the session.

Working

  • Firewall maintains a state table
  • Monitors connection status (new, established, terminated)
  • Allows only packets that belong to valid sessions
  • Blocks unsolicited or suspicious traffic

Benefit:
Provides better security than simple packet filtering.

16. What are VPN and secure tunneling? Explain briefly. Explain the operation of a VPN and how it ensures secure communication over the Internet.

VPN (Virtual Private Network)

A VPN creates a secure encrypted connection over a public network, allowing private communication.

Secure Tunneling

Secure tunneling encapsulates data packets within encrypted tunnels to protect them from interception.

Operation of a VPN

  1. User initiates VPN connection
  2. Authentication is performed
  3. Encrypted tunnel is established
  4. Data is encrypted before transmission
  5. Secure data transfer occurs over the Internet

How VPN Ensures Security

  • Encrypts data
  • Hides IP address
  • Prevents eavesdropping
  • Ensures confidentiality and integrity
