19. Interpret how OSPF uses Dijkstra’s algorithm for routing. What is DNS? Describe how it works.

OSPF and Dijkstra’s Algorithm

OSPF is a link-state routing protocol that uses Dijkstra’s Shortest Path First algorithm. Routers build a complete topology map and compute the shortest path based on cost.

How OSPF Uses Dijkstra’s Algorithm

  • Each router using OSPF collects information about its directly connected links.
  • Routers exchange this information using Link State Advertisements (LSAs).
  • All routers build an identical Link State Database (LSDB) representing the network topology.
  • OSPF then applies Dijkstra’s Shortest Path First (SPF) algorithm on the LSDB.
  • The algorithm calculates the shortest path based on cost (metric) from the router to every other network.
  • The best paths are stored in the routing table and used for packet forwarding.

DNS (Domain Name System)

DNS translates human-readable domain names into IP addresses, allowing users to access websites easily.

Example: www.google.com → 142.250.190.14

Without DNS, users would need to remember IP addresses instead of domain names.

Working of DNS

  • The user enters a domain name in a web browser.
  • The request is sent to a DNS resolver (usually provided by the ISP).
  • If the resolver does not know the answer, it queries:
    • Root DNS server
    • Top-Level Domain (TLD) server (e.g., .com, .org)
    • Authoritative DNS server
  • The authoritative server returns the corresponding IP address.
  • The resolver sends the IP address back to the user’s device.
  • The browser connects to the destination server using that IP address.

Network Switch

A network switch is a networking device used to connect multiple devices within a Local Area Network (LAN) and forward data frames to the correct destination using MAC addresses. It operates mainly at the Data Link Layer (Layer 2) of the OSI model (some switches also operate at Layer 3).

    Uses a MAC address table to send data only to the intended device.

    Allows many devices such as computers, printers, and servers to connect.

    Each port is a separate collision domain, reducing network congestion.
  4. Full-Duplex Communication
    Supports simultaneous sending and receiving of data.
  5. High Speed
    Supports Fast Ethernet, Gigabit Ethernet, and higher speeds.
  6. VLAN Support (Managed Switches)
    Allows logical segmentation of a network for better security and management.

The Transport layer is the fourth layer of the OSI model and is responsible for end-to-end communication between source and destination devices.

Functions of the Transport Layer

  1. Segmentation and Reassembly
    Breaks large data into smaller segments and reassembles them at the destination.
  2. Flow Control
    Regulates the speed of data transmission to avoid receiver overload.
  3. Error Control
    Detects errors and ensures retransmission of lost or corrupted data.
  4. Reliability
    Ensures complete and correct data delivery using acknowledgements and sequencing.
  5. Port Addressing
    Uses port numbers to deliver data to the correct application.
  6. Connection Management
    Establishes, maintains, and terminates connections.

24. Describe any two routing algorithms and their working. Evaluate the PVR (Priority-based Vector Routing) algorithm and explain its advantages.

Routing Algorithms

A routing algorithm determines the best path for data packets to travel from a source to a destination across a network. Routing decisions are based on metrics such as distance, cost, delay, bandwidth, or priority.

1. Distance Vector Routing Algorithm

Working:

  • Each router maintains a routing table containing the distance (metric) to each destination and the next hop.
  • Routers periodically exchange routing tables with neighboring routers.
  • Based on received updates, routers recalculate the shortest path.
  • Common metric: hop count.

Example: RIP (Routing Information Protocol)

Features:

  • Simple to implement
  • Uses periodic updates
  • Slower convergence and prone to routing loops

2. Link State Routing Algorithm

Working:

  • Each router discovers its direct neighbors and link costs.
  • Routers broadcast link-state information to all other routers.
  • Every router builds a complete network topology map.
  • The shortest path is calculated using Dijkstra’s algorithm.

Example: OSPF (Open Shortest Path First)

Features:

  • Fast convergence
  • Loop-free routing
  • Higher memory and processing requirements

PVR (Priority-based Vector Routing) Algorithm

PVR (Priority-based Vector Routing) is an enhanced routing algorithm that considers priority levels of data along with routing metrics to determine the best path.

Working of PVR Algorithm

  • Each data packet is assigned a priority value (high, medium, or low).
  • Routers maintain routing information along with priority vectors.
  • During congestion, high-priority packets are routed first.
  • Routing decisions consider both path cost and packet priority.
  • Low-priority traffic may be delayed or rerouted to avoid congestion.

Advantages of PVR Algorithm

  1. Quality of Service (QoS) Support
    Ensures faster delivery of high-priority data such as voice and video.
  2. Efficient Traffic Management
    Reduces congestion by prioritizing critical packets.
  3. Improved Network Performance
    Optimizes bandwidth utilization.
  4. Flexibility
    Suitable for real-time and multimedia applications.
  5. Reliability
    Critical data reaches the destination with minimal delay.

Types of Network Devices and Their Functions

Network devices are hardware components used to connect, manage, and control communication within a computer network.

  1. Hub
    • A basic networking device that connects multiple computers in a LAN.
    • Broadcasts data to all connected devices.
    • Operates at the Physical Layer (Layer 1).
  2. Switch
    • Connects devices within a LAN and forwards data based on MAC addresses.
    • Reduces collisions and improves performance.
    • Operates at the Data Link Layer (Layer 2).
  3. Router
    • Connects different networks and forwards data using IP addresses.
    • Determines the best path for data transmission.
    • Operates at the Network Layer (Layer 3).
  4. Modem
    • Converts digital signals into analog and vice versa.
    • Used to connect a network to the Internet.
  5. Bridge
    • Connects two similar networks and filters traffic using MAC addresses.
    • Reduces network congestion.
  6. Gateway
    • Acts as an interface between different network protocols.
    • Used in large and heterogeneous networks.

Distance Vector Routing (DVR) Algorithm

The Distance Vector Routing (DVR) algorithm is a routing technique in which each router maintains a routing table (distance vector) containing:

  • Destination network
  • Distance (metric such as hop count)
  • Next hop

Working of DVR Algorithm

  • Each router initially knows only its direct neighbors.
  • Routers periodically exchange routing tables with their neighbors.
  • Based on received information, routers update their routing tables.
  • The best route is selected using the minimum distance metric.
  • This process continues until all routers have consistent routing information.

Example: RIP (Routing Information Protocol)

Count-to-Infinity Problem in DVR

The count-to-infinity problem is a major drawback of the DVR algorithm.

Explanation

  • Occurs when a network link fails.
  • Routers slowly increase the distance metric to a destination, believing an alternative path exists through neighboring routers.
  • The metric value keeps increasing until it reaches a predefined maximum value (called infinity, e.g., 16 in RIP).
  • During this time, routing loops and incorrect routing occur.

Effects

  • Slow convergence
  • Increased network traffic
  • Temporary routing loops

Solutions

  • Split Horizon
  • Poison Reverse
  • Hold-down Timers

Routers are networking devices that forward data packets between networks. Based on their position and function in a network, routers are mainly classified as core routers and edge routers.

Roles of Core Routers

  • Forward large volumes of data at very high speed
  • Operate within the backbone of ISPs or large enterprise networks

  • Ensure low latency and high reliability

Core routers usually do not connect directly to end users.

Roles of Edge Routers

Edge routers are placed at the boundary of a network and connect internal networks to external networks, such as the Internet.

  • Act as the entry and exit point of a network
  • Connect LANs to WANs or ISPs
  • Perform routing, filtering, NAT, and security policies
  • Control traffic entering and leaving the network

Edge routers interact directly with end-user networks.

BGP (Border Gateway Protocol)

BGP (Border Gateway Protocol) is a path-vector routing protocol used for inter-domain routing, i.e., routing between different Autonomous Systems (AS) on the Internet.
It is the primary routing protocol of the Internet.

How BGP Works

  1. Each network on the Internet is identified as an Autonomous System (AS) with a unique AS number.
  2. BGP routers establish a TCP connection (port 179) with neighboring AS routers.
  3. Routers exchange routing information in the form of BGP update messages.
  4. Instead of shortest path, BGP selects routes based on:
    • AS path length
    • Routing policies
    • Next-hop attributes
  5. The best path is selected and stored in the BGP routing table.
  6. This process enables data packets to travel across multiple ISPs and networks efficiently.

Mobile IP

Mobile IP is a protocol that allows a mobile device to move from one network to another while maintaining the same IP address, ensuring uninterrupted communication.

Mobility Management in Mobile IP

How Mobile IP Works

  1. When the mobile node is in its home network, it communicates normally.
  2. When it moves to a foreign network, it obtains a Care-of Address (CoA).
  3. The mobile node registers the CoA with its Home Agent.
  4. The Home Agent intercepts packets meant for the mobile node.
  5. Packets are tunneled from the Home Agent to the Care-of Address.
  6. The Foreign Agent forwards packets to the mobile node.

Benefits of Mobility Management

  • Seamless communication while moving
  • No change in IP address for applications
  • Continuous sessions for calls and data transfer

RIP (Routing Information Protocol)

RIP (Routing Information Protocol) is a distance vector routing protocol used to determine the best path for data packets in a network based on hop count.

How RIP Works

  1. Each router maintains a routing table containing destination networks, hop count, and next hop.
  2. Routers exchange routing tables periodically (every 30 seconds) with neighboring routers.
  3. RIP uses hop count as the routing metric.
  4. The path with the least number of hops is chosen as the best route.
  5. The maximum hop count is 15; a hop count of 16 is considered unreachable.
  6. When a network change occurs, routers update their routing tables accordingly.

Features of RIP

  • Simple and easy to implement
  • Suitable for small networks
  • Uses distance vector algorithm
  • Slow convergence
  • Not suitable for large or complex networks
